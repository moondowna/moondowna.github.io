vue

1. 一套用于搭建用户界面的渐进式js框架

2. 特点：

  • 采用组件化模式，提高代码复用率，更好维护
  • 声明式编码，提高开发效率
  • 虚拟dom+diff算法，复用dom节点
  • diff算法：只能平级比较，通过比较key值，若Key值相同比较后面的内容，若不同则向下比较

3. 属性–容器与vue实例一一对应

  • el：用于指定当前为特定容器服务的vue实例，value为字符串 或者 document.getElement…也可以
  • v.$mount指定容器
  • data：写一个对象，用于存储数据，供el所指定的容器去使用
    • 插值语法：{ { key } }，key必须为js里的表达式（会有返回值 a、a+b、fn(2)..）
      往往用于指定标签体内容,可以读取data属性b
    • 指令语法：
      • v-bind绑定
        <a v-bind:herf=' url '> 执行表达式，将结果绑定给herf
        简写： v-bind ==== :
        <a : herf=' url '>
        用于解析标签（标签属性、内容、绑定事件…）
    • data写法–>对象式&函数式：
      • 对象式：
        data:{ name:ah}
      • 函数式：
        data:function(){ return {name:ah } } 此处运用箭头函数，则this指向window，普通实例函数指向实例对象

4. 单向绑定与双向绑定：

  1. v-bind单向绑定:数据只能从data流向页面
  2. v-model双向绑定：能从data流向页面，也能从页面流向data
    只能应用于表单类元素（输入类元素，input，select）
    v-model:value可以简写为v-model

5. mvvm模型：

  • m: mode : data中的数据
  • V: view : 模板代码
  • Vm: 视图模型
  • data中所有的属性都出现在vm中保存，vm还有一些vue原型上的属性，在vue模板中可以直接使用

6. 数据代理：setter&getter

  • 数据代理：通过vm对象来代理data对象中属性的读写操作
    vm._data==
  • 好处：更简洁
  • 原理：通过Object.definePropert( )把data对象中所有属性添加到vm上。getter为每一个添加到vm上的属性指定一个getter/setter

7. 事件

  1. v-on:click = ‘ 事件名 ‘，事件写在vue的methods对象中，最终会在vm上
  2. @xxx ，xxx也可以是一些简单的执行语句。
  3. methods中配置的函数，不要用箭头函数，否则this就不是vm了
  4. 简写： @click = ‘ 函数名 ‘
  5. 参数传递：在函数名后面添加小括号
    @click = ' showInfo ($event,6) ' methods:{ showInfo ( event , number ) }
    其中event默认就有 –> @click='xxx' == @click='xxx($event)'
  6. 事件修饰符：
  • .prevent 阻止默认事件 == e.prenventDefult( )
  • .stop 阻止事件冒泡 == e.stoppp 先捕获后冒泡
  • .once 事件只触发一次
  • .capture 先使用事件的捕获模式
  • .self 只有e.target是当前操作元素才触发事件
  • .passive 事件的默认行为立即执行，无需等待事件回调执行完毕
  1. 键盘事件：
  • 回车 => enter
  • 删除 => delete
  • 退出 => esc
  • 空格 => spcae
  • 换行 => tab
  • 上下左右 => up down left right
  • vue.config.keycodes自定义键名

8. 常用指令

  1. v-text:设置标签的内容–>内容绑定
    默认写法会替换全部内容（使用插值表达式{ }可以替换指定内容）
    只会解析成文本
  2. v-html：设置元素的innerHtml–>内容绑定
    可以识别标签
  3. v-on:为元素绑定事件，简写为@ –>事件绑定
    绑定方法存在于methods中（写在data中也能实现，mvvm）
    方法内部通过this访问在data中的数据
  4. v-show:根据表达式的真假切换元素的显示状态
    应用于广告，遮罩层
    原理：display:none
  5. v-if:根据表达式的真假切换元素的显示状态
    里面的值为true则存在于dom树中显示元素，为false则移除且隐藏元素
    原理：直接操作dom，消耗性能较大
  6. v-bind:设置元素的属性(src.title,class) 简写为 :
    <img :src=' ' :class = { active:isActive } >
  7. v-for:根据数据生成列表结构：
    <li v-for=' ( item,index) in arr :key="index">
    item:每项 index：索引值 arr:可以是数组、对象、字符串、指定次数
    item和index可以结合其他指令

9. 计算属性：computed对象中 必须有返回值

  1. 定义:要用的属性不存在，通过已有属性计算得来
  2. 原理：借助objec.defineproperty提供的getter和setter
  3. get：必须写 是对象
    执行：当初次读取数据或依赖的数据发生改变时
  4. set:可以不用写 是对象
    执行：计算属性要被修改，写在set里响应修改
  5. 优点：与methods相比，内部有缓存机制，效率更高，调试方便。
  6. 简写：当只有get没有set时，可省略为函数（本质不是函数）
    fullname(){ };

10. 监视属性 watch

  1. 当监视的属性变化时，回调函数自动调用
  2. 书写方式：
    watch:{ }
    或者 vm.$watch ( 'key' ,{ } )
  3. 回调函数：
  • handler( newvalue,oldvalue)
  • 当检测的属性值发生改变时调用
  • Immediate: 默认为false 初始化时默认handler不被调用
  1. 深度监视：
  • 监视多级结构中某个属性的变化： " number.a "
  • 监视多级结构中所有属性的变化：Deep: true
  1. computed与watch区别：
  • computed能完成的功能，watch都能完成
    1. watch还可以完成异步任务 setTimeout
    2. 被vue管理的函数，最好是普通函数，这样this的指向才是实例化对象
    3. 不被vue管理的函数（定时器，ajax回调）最好写成箭头函数，这样this才指向实例化对象

11. 绑定class样式：

  1. 字符串写法： 确定的样式正常写
    不确定的样式 :class=' mood '
    适用于：样式的类名不确定，需要动态指定
  2. 数组写法： :class=' classarr '
    适用于：需要绑定的样式个数不确定，名字也不确定
  3. 对象写法：
    适用于：要绑定的样式个数确定，名字确定，但要动态指定用不用

12. 绑定style样式：

  1. :style=' { fontSize：“40px”} '
  2. :style=" [a,b]" 其中a,b均为样式对象

13. 条件渲染：

  1. V-show=“ ” 变换频繁使用
    原理：display:none
    可以直接等于布尔值，也可以是表达式（根据表达式的结果返回布尔值)

  • V-else-if 在执行完第一个if之后不再执行else内容

  3. 列表排序

15. 生命周期（生命周期钩子）：

  1. 常用的生命周期钩子：
  2. mounted：发送ajax请求、启动定时器、绑定自定义事件、订阅消息等初始化操作
  3. beforeDestroy：清除定时器、解绑自定义事件、取消订阅消息等收尾工作
  4. 销毁vue：
  • 销毁后借助vue开发工具看不到任何消息
  • 销毁后自定义事件会失效，但原生dom事件依然有效
  • 一般不会再beforeDestroy操作数据，因为即便操作数据，也不会再触发更新流程了

16. 组件：实现应用中局部功能代码和资源的集合

  1. 传统方式：依赖关系混乱，不好维护。代码复用率不高 组件：复用编码、简化项目编码、提高运行效率
  2. 模块化、组件化
  3. 组件步骤：
  • 创建组件: const a = Vue.extend( { } )
  • 注册组件：局部注册：compnents:{ h:a }内部使用键值对
  • 全局注册：Vue.component ( ‘ ‘ , )
  • 编写组件标签
    1. 组件嵌套：components{}
    2. 组件本质：构造函数
    3. 单文件组件：app.vue、index.html、main.js
    4. 组件传值
    • 子传父props：
      只接收：props:[ " name " ]
      限制类型：props:{ age:Number}
      限制类型+限制必要性+指定默认值：
      props:{ name:{ type:String, //类型 required:true, //必要性 default:' '//默认值 }
      }

17. 脚手架：

  1. 项目结构：
  • favicon.ico:网站页签图标
  • index.html：html文件
  • src/assets：存放静态资源
  • components：存放所有组件
  • App.vue:所有组件的父组件
  • main.ji：首先运行
  • .gitignore:不想接受git管理的文件
  1. vue.js与vue.runtime.xxx.js的区别：
  • vue.js是完整版的vue，包含核心功能+模板解析器
  • vue.runtime.xxx.js是运行版的vue，只包含核心功能，没有模板解析器
