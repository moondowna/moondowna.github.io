暴露与引用：

暴露：

  • 分别暴露：在每个数据前加 export
  • 统一暴露： export { }
  • 默认暴露（常用）： export default { }

引用：

  • 通用引入:import * as .... form..
  • 对于默认暴露 :import ... from ...
    其他: import { ... 可以解构赋值} from ...

脚手架：

项目结构：

  1. favicon.ico:网站页签图标
  2. index.html：html文件
  3. src/assets：存放静态资源
  4. components：存放所有组件
  5. App.vue:所有组件的父组件
  6. main.ji：首先运行
  7. .gitignore:不想接受git管理的文件
  8. vue.js与vue.runtime.xxx.js的区别：
  • vue.js是完整版的vue，包含核心功能+模板解析器
  • vue.runtime.xxx.js是运行版的vue，只包含核心功能，没有模板解析器

组件自定义事件：

  1. 用于子组件给父组件传值
  2. app.vue 中
  3. <student @ah='foo()',foo在vue中写，ah组件自定义事件在student组件中写
  4. this.$.emit ( ah )
  5. 解绑事件：$.destory()

ref属性，相当于id：

  • 用来给元素或子组件注册引用信息
  • 添加:<h1 ref = " xxx "> </h1>
  • 获取: this.$refs.xxx
  • 在组件标签上得到i的是组件实例对象vc
  • 在html标签上得到的是真实dom元素

scoped样式

  1. 作用:style样式在当前的组件中生效，防止在全局发生冲突
  2. 使用:<style scoped>

全局事件总线,在任意组件间通信，尤其是兄弟组件：

  1. 安装全局事件总线
  2. 接受数据 this.$bus.$on ( " " , 数据 ),一般在mouted注册
  3. 提供数据this.$bus.$emit( " " , 数据 )
  4. 解绑事件，在beforeDestory中，用$off解绑

插槽：

  1. 作用：让子组件在父组件指定位置插入html结构，也是一种组件间通信的方式，适用于父组件给子组件。
  2. 在组件间书写添加的html结构
  3. 默认插槽
  4. 具名插槽
  5. 作用域插槽

Vuex

  • Action指代用户动作所产生的新数据
  • Dispatcher在Vuex中有Actions和Mutations两种实现，Actions应用于对Mutations的包装
  • Store为不应被除Mutations以外任何方法更新的数据存储
  • View的更新则由Vue.js完成

使用：

  1. 创建store/index.ji
  2. 在main.js中创建vm时传入store配置
  3. 若没有网络请求或其他业务逻辑，组件中也可以越到actions，不用写dispatch

路由

  1. 路由组件通常存放在pages文件夹中，一般组件放在components文件夹中
  2. 通过切换，隐藏了的路由组件，默认被销毁，需要的时候再去挂载
  3. 每个组件都有自己的$route属性，里面存储着自己的路由信息
  4. 整个应用只有一个router，通过组件的$router属性获得
  5. 注册
  6. 路由方法
    to切换:<router-link active-class='active' to="/xxx"></router-link>
  7. 相对路径与绝对路径
  8. 嵌套路由 children
    二级路由不加/
